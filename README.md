# DevKeeper — Управление проектами и задачами

**DevKeeper** — это гибкое десктопное приложение для организации любых проектов и задач. Создавайте проекты, разбивайте их на задачи и подзадачи, отслеживайте статусы, используйте шаблоны и быстро находите нужное.

> 📌 Это **сетевая версия** для совместного использования. Все данные хранятся на центральном сервере, а клиенты подключаются к нему через локальную сеть.

---

## 🚀 Возможности

- **Проекты** – создавайте проекты для любых целей (работа, учёба, дом, хобби). Для проектов с кодом можно указать путь к папке – тогда при добавлении автоматически распарсятся технологии из файла `requirements.txt` (они появятся в виде тегов и будут участвовать в поиске).
- **Задачи** – неограниченная вложенность, описание, заметки, сниппеты кода (если нужно).
- **Статусы** – `Новое`, `В процессе`, `Готово`, `Архивное`. Статус родительской задачи обновляется автоматически в зависимости от статусов подзадач.
- **Избранное** – закрепляйте важные проекты и задачи вверху списка.
- **Шаблоны** – создавайте готовые шаблоны задач и добавляйте их в проект одним кликом.
- **Поиск** – быстрый поиск по названию проекта и технологиям.
- **Горячие клавиши** – ускоряйте работу: `Ctrl+P` (новый проект), `Ctrl+T` (новая задача), `Ctrl+=` (развернуть все задачи), `Ctrl+-` (свернуть все задачи), `Delete` (удаление).
- **Темы** – светлая, тёмная, системная.
- **Интеграция с PyCharm** – при указании пути к исполняемому файлу можно открывать проект прямо из приложения (полезно для разработчиков).
- **Совместная работа** – несколько пользователей могут видеть и редактировать общие проекты.
- **Приватность** – проекты и задачи могут быть личными (только для автора) или общими для всех.
- **Управление пользователями** – каждый пользователь выбирает своё имя при входе.

## 📦 Установка

1. Скачайте установщик `DevKeeper_Setup.exe` со страницы [Releases](https://github.com/Tenrai-chi/dev_keeper/releases).
2. Запустите установщик и следуйте инструкциям.
3. На этапе выбора компонентов укажите:
   - **DevKeeper Client** – если вы будете подключаться к серверу.
   - **DevKeeper Server** – если вы будете запускать сервер на своей машине.
   - Можно установить оба компонента для тестирования на одном компьютере.
4. После установки запустите сервер (если вы его установили) и клиент.

> При первом запуске клиент попросит ввести адрес сервера (например, `http://192.168.1.100:8888/api`). Адрес сервера выводится в консоли при его запуске.

---

## 🖥️ Запуск сервера

Сервер запускается через ярлык в меню «Пуск» или из папки установки. При запуске в консоли отображается локальный IP-адрес и порт (по умолчанию 8888). Используйте этот адрес для настройки клиентов.

Если нужно изменить хост или порт, запустите сервер с параметром, например:
  ```bash
  DevKeeperServer.exe --host 192.168.0.2 --port 9000
  ```

---
## 🛠️ Для разработчиков

- Сборка клиента в .exe:
    ```bash
    pyinstaller client.spec
    ```
- Но также можно просто запустить файл .spec:
    ```bash
    pyinstaller server.spec
    ```

## 📂 Структура данных
- Клиент – данные хранятся в %APPDATA%\DevKeeperClient\ (база данных projects.db, шаблоны default_tasks.json, логи).
- Сервер – данные хранятся в %APPDATA%\DevKeeperServer\ (база данных network_projects.db, логи).
- При удалении приложения программа спросит, хотите ли вы удалить и пользовательские данные.
